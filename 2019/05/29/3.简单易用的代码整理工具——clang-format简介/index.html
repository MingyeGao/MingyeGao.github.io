代码格式整理工具——clang-format简介

最近教研室有好多陈旧的代码要整理，借此介绍一款简单易用的代码格式规范工具。

clang-format 功能简介

clang-format是一款用于整理代码格式的自动化工具。主要适用的情况有：

  • 团队共同开发代码，不同人的代码习惯不同，难以统一，需要以一种简单可行的工具对代码格式进行统一
  • 手上的代码格式比较混乱，需要一种快速的方式规范代码格式，增强程序可读性

遇到以上情况可以使用clang-format，快速高效地改进代码格式

.clang-format中的每一项在clang-format style 官方文档都有具体的说明，大多数说的比较清楚，还有具体例子。不过几个Penalty开头的选项讲得比较含糊，也没有举例子

  • PenaltyBreakAssignment (unsigned): The penalty for breaking around an assignment operator.
  • PenaltyBreakBeforeFirstCallParameter (unsigned): The penalty for breaking a function call after call(.
  • PenaltyBreakComment (unsigned): The penalty for each line break introduced inside a comment.
  • PenaltyBreakFirstLessLess (unsigned): The penalty for breaking before the first <<.
  • PenaltyBreakString (unsigned): The penalty for each line break introduced inside a string literal.
  • PenaltyBreakTemplateDeclaration (unsigned): The penalty for breaking after template declaration.
  • PenaltyExcessCharacter (unsigned): The penalty for each character outside of the column limit.
  • PenaltyReturnTypeOnItsOwnLine (unsigned): Penalty for putting the return type of a function onto its own line.

这里的Penalty可以理解为“代价”， “代价”的数值越大，则越不倾向于做某事。例如PenaltyBreakAssignment可以理解为“在赋值后换行的代价”。有时一行的长度超过了ColumnLimit的长度，但是换行可能会影响观感，此时就会比较相对应的“代价因子”，判断是否要换行。举个例子说明一下：

字符串所在行的长度超过了ColumnLimit 80，此时可以在字符串中间折断换行，也可以将整个字符串挪到下一行。

当 PenaltyBreakString=0 时，字符串中间引入每个换行符的代价小，因此可以中断并引入下一行；当 PenaltyBreakString=10000 时，字符串中间引入每个换行符的代价大，因此clang-format选择将整个字符串挪到下一行，不在字符串中间折断换行。

需要注意的是，代价因子只作一种定性的判断依据，无法通过“代价因子”的数值直接计算得出换行与否。
