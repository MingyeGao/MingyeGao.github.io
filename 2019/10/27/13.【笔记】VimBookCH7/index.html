Vim Book Chapter7: Command for Programmers

备忘笔记

设置缩进字符数

:set shiftwidth=4

自动缩进选项

三种模式

cindent Vim自动根据c编程风格缩进

smartindent 每一行和前一行缩进相同，有{时多缩进一级，有}时少缩进一级

autoindent 新的一行与前一行缩进相同

大括号内容自动缩进

光标移动到第一个左大括号上，输入 =% ，大括号中的内容自动缩进

跳转

gd gD 查找变量的声明

(注：VimBook 表示 gd 用于查找局部变量声明， gD 用于查找全局变量申明，但在本例中二者没有区别)

e.g.

将光标放在 main 中的 global_var 上， gD 可以转到第一行申明处

将光标放在 main 中的 local_var 上， gd 可以跳转到函数内的声明上

[ \ , ] \ 在当前文件和include的文件中，查找当前变量

两者在执行结果上有区别，可能跟操作系统与vim版本有关

e.g.

在 code2.c 中，光标移动到 header_var 上， [ ctrl-I ，会跳转到 code2.h 中 header_var 的定义上。

不只是变量声明，也可以跳转到函数定义

[ \<ctrl-D>, ] \<ctrl-D> 跳转到宏定义

光标移动到 for 循环中的 SIZE 上

[ <ctrl+D> 跳转到 #define SIZE 10 上

] <ctrl+D> 跳转到下面的 #define SIZE 20 上

[d ]d 在屏幕下方显示宏定义

[d 向上查找， ]d 向下查找

匹配

% 可以匹配:

  • () [] {}
  • #ifdef #endif
  • #ifndef #endif
  • #if #else #endif

用 VISUAL MODE 缩进

用 v 进入 VISUAL 模式，选择需要缩进的内容

然后 >

查找一个命令的man 页面

将光标停留在命令上， K

不同系统的手册可能不一样，还可能会带有参数

标签(tags)

定义：函数定义的位置，被称作tag

创建标签

默认会在当前目录下创建一个名为 tags 的文件，存储代码的函数符号信息

用 -f 可以指定生成的文件名

如果不指定，新生成的 tags 会覆盖原 tags

跳转到指定函数的定义

:tag [function]

函数调用间的跳转

e.g

现有函数调用关系：

writeBlock()->writeLine()->writeChar()

从任意位置跳转到 writeBlock() 定义： :tag writeBlock()

在 writeBlock() 中想查看 writeLine() 的定义： 光标转到 writeLine(), <ctrl+]>

回到 writeBlock() ： <ctrl+T>

若函数定义在不同文件中，也可以互相跳转，但要针对整个路径生成 tags 文件

执行 ctags -R 指令后，各文件的函数符号信息都被记录了。

分屏并显示函数信息

:stag [function]
